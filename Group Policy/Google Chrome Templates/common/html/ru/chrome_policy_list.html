Главная страница

Настраивает домашнюю страницу по умолчанию в Google Chrome и запрещает пользователям изменять ее. Пользовательские настройки главной страницы полностью блокируются, если в качестве главной страницы выбрана страница быстрого доступа или указан определенный URL. Если URL главной страницы не указан, пользователь может установить страницу быстрого доступа в качестве главной страницы, набрав URL chrome://newtab.
В начало

HomepageLocation

Настройка URL домашней страницы
Тип данных:
String (REG_SZ)
Расположение в реестре Windows:
Software\Policies\Google\Chrome\HomepageLocation
Предпочтительное название для Mac и Linux:
HomepageLocation
Поддерживается в:
  • Google Chrome (Linux, Mac, Windows) начиная с версии 8
  • Google Chrome OS (Google Chrome OS) начиная с версии 0.11
Поддерживаемые функции
Рекомендуется: Да, Динамическое обновление правил: Да
Описание:
Задает URL главной страницы по умолчанию в Google Chrome и запрещает пользователям его менять. Главная страница открывается с помощью одноименной кнопки. Страницы, открываемые при запуске, определяются правилами RestoreOnStartup. На главной странице может открываться заданный здесь URL или страница быстрого доступа. Во втором случае это правило игнорируется. Если этот параметр включен, пользователи не могут менять URL главной страницы в Google Chrome, но могут установить страницу быстрого доступа в качестве главной. Если это правило не установлено, пользователи могут выбрать главную страницу по своему усмотрению (при условии, что правило HomepageIsNewTabPage также не определено).
Пример значения:
"http://chromium.org"
В начало

Сделать страницу быстрого доступа главной
Тип данных:
Boolean (REG_DWORD)
Расположение в реестре Windows:
Software\Policies\Google\Chrome\HomepageIsNewTabPage

HomepageIsNewTabPage
Поддерживается в:
  • Google Chrome (Linux, Mac, Windows) начиная с версии 8
  • Google Chrome OS (Google Chrome OS) начиная с версии 0.11
Поддерживаемые функции
Рекомендуется: Да, Динамическое обновление правил: Да
Описание:
Настраивает тип главной страницы в Google Chrome и запрещает пользователям менять ее настройки. В качестве главной страницы можно выбрать определенный URL или пустую вкладку. Если этот параметр включен, в качестве главной страницы всегда используется пустая вкладка, а заданный URL главной страницы игнорируется. Если этот параметр отключен, пустая вкладка будет открываться только в том случае, когда в качестве URL главной страницы указан путь chrome://newtab. Когда этот параметр включен или отключен, пользователи не могут менять его значение в Google Chrome. 0x00000001 (Windows), true (Linux), <true /> (Mac)

Если выбран параметр "Продолжить с места разъединения", при запуске Google Chrome будут восстанавливаться вкладки, которые были открыты в момент последнего завершения работы.   • 5 = Открыть страницу быстрого доступа
  • 1 = Восстановить последний сеанс
  • 4 = Открыть одну или несколько страниц

Для каждого пункта Google Chrome найдет по идентификатору нужное расширение для службы обновления по указанному URL-адресу и автоматически установит его. Например, lcncmkcnkcdbbanbjakcencbaoegdjlp;https://clients2.google.com/service/update2/crx устанавливает расширение Google SSL Web Search, используя стандартный URL-адрес для обновления Интернет-магазина Chrome. Подробнее о хостинге для расширений см. http://code.google.com/chrome/extensions/hosting.html. Пользователи не смогут удалить расширения, которые указаны в этих правилах. Если исключить расширение из списка, Google Chrome автоматически удалит его. Если же эти правила не установлены, пользователь может самостоятельно удалять любое расширение из Google Chrome.

Если это правило настроено, Google Chrome использует указанный каталог, даже когда пользователь установил флаг командной строки --user-data-dir. Если это правило не настроено, используется путь в профиле по умолчанию, но пользователь может изменить его с помощью флага --user-data-dir.
Пример значения:
"${users}/${user_name}/Chrome"
