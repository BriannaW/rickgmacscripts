Administrador de contraseñas

Permite configurar el Administrador de contraseñas. Si el Administrador de contraseñas está habilitado, tienes la posibilidad de habilitar o inhabilitar la opción de visualización de contraseñas almacenadas de los usuarios sin encriptar.
Volver al principio

PasswordManagerEnabled

Habilitar el administrador de contraseñas
Tipo de datos:
Boolean (REG_DWORD)
Ubicación del registro de Windows:
Software\Policies\Google\Chrome\PasswordManagerEnabled
Nombre de preferencia en Mac/Linux:
PasswordManagerEnabled
Compatible con:
  • Google Chrome (Linux, Mac, Windows) desde la versión 8
  • Google Chrome OS (Google Chrome OS) desde la versión 0.11
Funciones admitidas:
Se puede recomendar: Sí, Actualización dinámica de política: Sí
Descripción:
Permite habilitar el almacenamiento de contraseñas y el uso de contraseñas guardadas en Google Chrome. Si habilitas esta opción, Google Chrome podrá memorizar las contraseñas de los usuarios, que aparecerán automáticamente la próxima vez que accedan a un sitio. Si inhabilitas esta opción, los usuarios no podrán guardar contraseñas ni utilizar contraseñas almacenadas. Independientemente de que habilites o inhabilites esta opción, los usuarios no podrán modificarla ni anularla en Google Chrome. Si no se establece esta política, esta opción estará habilitada, pero los usuarios podrán modificarla.
Valor de ejemplo:
0x00000001 (Windows), true (Linux), <true /> (Mac)

ExtensionInstallForcelist

Configurar lista de extensiones de instalación forzada
Tipo de datos:
List of strings
Ubicación del registro de Windows:
Software\Policies\Google\Chrome\ExtensionInstallForcelist
Nombre de preferencia en Mac/Linux:
ExtensionInstallForcelist
Compatible con:
  • Google Chrome (Linux, Mac, Windows) desde la versión 9
  • Google Chrome OS (Google Chrome OS) desde la versión 0.11
Funciones admitidas:
Actualización dinámica de política: Sí
Descripción:
Permite especificar una lista de extensiones que se podrán instalar automáticamente sin intervención del usuario. Todos los elementos de la lista son cadenas que contienen un ID de extensión y una URL de actualización delimitados por punto y coma (;). El ID de extensión es la cadena de 32 letras que se encuentra, por ejemplo, en chrome://extensions cuando está en modo de desarrollador. La URL de actualización debe apuntar a un documento XML de manifiesto de actualización, tal y como se describe en la página http://code.google.com/chrome/extensions/autoupdate.html. Para cada elemento, Google Chrome recuperará la extensión especificada por el ID de extensión del servicio de actualización en la URL de actualización especificada y la instalará automáticamente. Por ejemplo, lcncmkcnkcdbbanbjakcencbaoegdjlp;https://clients2.google.com/service/update2/crx instala la extensión Google SSL Web Search de la URL de actualización de Chrome Web Store estándar. UserDataDir
